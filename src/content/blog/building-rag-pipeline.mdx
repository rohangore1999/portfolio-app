## Introduction

Retrieval-Augmented Generation (RAG) is one of the most powerful patterns in modern AI engineering. Instead of relying solely on a model's pre-trained knowledge, RAG retrieves relevant documents at inference time and feeds them as context.

## Why RAG?

Large language models hallucinate. They also have a knowledge cutoff. RAG solves both problems by grounding responses in real, up-to-date documents.

```python
from langchain.vectorstores import Chroma
from langchain.embeddings import OpenAIEmbeddings

embeddings = OpenAIEmbeddings()
vectorstore = Chroma.from_documents(docs, embeddings)
retriever = vectorstore.as_retriever(search_kwargs={"k": 4})
```

## The Pipeline

1. **Ingestion** — Load documents, chunk them, embed with a model
2. **Storage** — Store embeddings in a vector DB (Chroma, Pinecone, Weaviate)
3. **Retrieval** — On query, embed the question and find top-k similar chunks
4. **Generation** — Feed retrieved chunks + query to the LLM as context

## Lessons Learned

- Chunk size matters more than you think — 512 tokens with 50-token overlap worked best for us
- Hybrid search (BM25 + vector) outperforms pure vector search on factual queries
- Reranking with a cross-encoder dramatically improves precision

## Conclusion

RAG is now a standard tool in the AI engineer's toolkit. The ecosystem has matured rapidly — pick your stack, start small, and iterate on retrieval quality.
